Form design

Admins can select corresponding fields to make forms according to the physical forms. User interfaces vary with field types. For instance, users can select one in several options when selection box is added by admins. User can only input numbers when ‚ÄúNumber‚Äù field is added. You can refer to ‚ÄúForm fields‚Äù for more details.

The following steps illustrate how to design a ‚ÄúReimbursement‚Äù form.

  • Click on ‚ÄúAdd Workflow‚Äù at the top of pop-up interface.
  • Click on ‚ÄúSettings‚Äù and then click ‚ÄúWorkflow Designer‚Äù after having entered the workspace.
  • Then enter the flow name ‚ÄúReimbursement‚Äù and click ‚ÄúOK‚Äù.
  • Click on ‚ÄúReimbursement‚Äù in the corresponding category. The pop-up interface will be ‚Äúworkflow‚Äù by default. Then click on ‚ÄúForm‚Äù at the top of the page to switch the interface.
  • Add Fields.
    • Click on ‚ÄúAdd Fields‚Äù in ‚ÄúForm‚Äù page. Then select ‚ÄúText‚Äù under ‚ÄúInput‚Äù and move it to appropriate place in the left region of the page.
    • Then select the previous ‚ÄúText‚Äù. Click on ‚ÄúProperties‚Äù and all properties of the ‚ÄúText‚Äù will be displayed. You can modify the properties if necessary. For instance, in the ‚ÄúReimbursement‚Äù form, we modify field name from ‚ÄúText‚Äù to ‚ÄúOrganization‚Äù. What‚Äôs more, you can input ‚Äú(applicant.organization.fullname)‚Äù in the formula box. After that, the system will find the organization automatically once an applicant submits a request. You can refer to ‚ÄúField Formula‚Äù for more details.

  • Then we should add more detailed information to the ‚ÄúReimbursement‚Äù form. So we add a ‚ÄúTable‚Äù field. The adding method is the same as adding a ‚ÄúText‚Äù field.
  • Next, we can modify the properties of ‚Äútable‚Äù field. All properties of the ‚ÄúTable‚Äù will be displayed after clicking on ‚ÄúProperties‚Äù. For instance, we can modify the field name from ‚ÄúTable‚Äù to ‚ÄúExpense Reports‚Äù. Input ‚Äúplease enter the actual use of all invoices‚Äù in description box.
  • Then we can add ‚ÄúExpense Category‚Äù, ‚ÄúAmount‚Äù, ‚ÄúDate‚Äù and ‚ÄúTotal Amounts‚Äù in the ‚ÄúExpense Report‚Äù.
    • Expense Category
      • It is a ‚ÄúDrop-down‚Äù field. We can modify its property. For instance, we can input several options in ‚ÄúList Values‚Äù box and select it as a required field.
    • Amount
      • It is a ‚ÄúNumber‚Äù field. We can modify its property. For instance, we can input its decimal places and default value. Except for that, we can select whether it is a required and wide field.
    • Date
      • Then we add a date filed and modify its field name to ‚ÄúThe date when the expense is incurred ‚Äù. Next, select it as a required field.
    • Remarks
      • It‚Äôs a text filed and we modify its name to‚ÄúRemarks‚Äù.
    • Total Amounts
      • It‚Äôs a number field and we modify its name to ‚ÄúTotal Amounts‚Äù. Next we input ‚Äúsum ({Amount})‚Äù in formula box. Then the system will automatically calculate the total amount when an applicant submits a reimbursement request.
  • Click the ‚Äú-‚Äù icon to delete the field.
  • Drag the field to move its location.
  • Save the form by click ‚ÄúSave‚Äù, and then click ‚ÄúClose‚Äù.
  • Return to workflow designer interface.
  • Run the flow.

If steps above are finished, then initiators can submit reimbursement requests.
