Docker 部署

本教程以 steedos-project-oa为例，指导你如何将项目编译成镜像发布至docker hub并使用 docker-compose 启动项目。

#准备工作

注册docker账号，如：exampleUsername

安装 docker命令行工具， 根据操作系统选择安装

终端登录 docker 账号：

docker login

安装docker-compose

#克隆项目

请访问 https://github.com/steedos/steedos-project-oa 先在项目主页右上角点 Fork 项目，然后将已经 fork 到自己账号下的项目 clone 到本地，以便提交修改：

#发布过程

进入本地 steedos-project-oa 项目目录：

cd steedos-project-oa

浏览器访问docker hub，并使用刚刚注册的账号登录，点击Create Repository按钮：

新建 Repository 名称如：steedos-project-oa

编译镜像：

docker build -t exampleUsername@steedos-project-oa:tagname

tagname 可以是 1.0、2.0 等自定义可明确表示版本的字符，build 命令文档

将镜像推送至 docker hub：

docker push exampleUsername@steedos-project-oa:tagname

推送完成之后可在docker hub页面看到镜像

可使用docker pull exampleUsername@steedos-project-oa:tagname拉取镜像到本地

#使用 docker-compose 启动服务

#启动数据库服务

使用 mongodb 官方镜像启动数据库服务，例如：

docker run --name mongo -d mongo:4.2

#启动应用服务

进入steedos-project-oa项目目录，配置好.env.local文件启动应用服务：

docker-compose up -d

配置.env.local 的目的是服务于steedos-config.yml
